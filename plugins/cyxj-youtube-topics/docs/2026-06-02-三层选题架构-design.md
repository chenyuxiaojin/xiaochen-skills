# 选题流水线重构：全网 / 白名单 / 领头羊榜 三层

日期：2026-06-02
状态：已与用户对齐，待实现

## 背景 / 触发的 bug

2026-05-30 起，每日选题报告被低质短视频灌爆（<5min 视频从 0–5 条/天 暴增到 20–37 条/天），
且用户自己白名单频道的视频反而看不见了。根因排查：

1. **时长豁免**（2026-05-12，commit 8ea27c3）+ **关键词豁免**（2026-05-29，commit 1b8ffb0）
   两道门同时对「信任频道」敞开 → 信任频道的 30–60 秒 shorts、跑题内容无门槛涌入。
   （shorts 这一刀已先修：MIN_DURATION 5→3min、移除时长豁免，commit 6dd85cc。）
2. **自动晋升门槛过松**：一条视频均播放≥5000 就 is_quality → 晋升为信任频道（仅需召回过 1 条），
   且晋升即获「直查该频道近 20 条 + 豁免关键词」特权 → 一条爆款就把一个频道的整批水文翻进报告。

## 用户的心智模型（教学比喻）

- **全网日常抓取** = 批改全班 50 个学生的作业（全看，不是只看几个）。发现的根基，**不动**。
- **白名单（助教）** = 用户亲手指定、信得过的 8 个种子频道。用户大部分时间看这个。
- **领头羊榜（晋升）** = 从全班作业里，凭**长期、反复**交出好作业累积出的「优等生排行」。
  用途是让用户一眼看到「谁值得收编成助教」，由用户**手动**加白名单。

核心纠偏：**「上榜」≠「发抓取特权」**。领头羊榜只是排名展示，上榜不该让系统去额外翻该频道库存。

## 三层设计

### 一、报告结构（从上到下）
1. **📌 白名单本期视频（助教）**：8 个种子频道本期所有 ≥3min 视频，单列最前，永不被淹。
2. **⬆️ 领头羊榜（Top 5）**：排名 + 上榜理由（发了几条 / 最近多活跃 / 均播放 / 首发次数）。
   对**未在白名单**的频道，加 **🔥 建议收编** 提示（被动展示 + 主动提示，两者都要）。
3. **全网话题主体（💎/👀/🔁/📋）**：维持不变。

### 二、根本改动：上榜不再发特权
- **自动晋升频道不再被直查、不再豁免关键词** → 只通过全网关键词召回出现（和所有人一样）。
- **领头羊榜 = 纯排名展示**，从创作者索引算出来，上榜不触发任何额外抓取。
- **只有 8 个白名单（种子）保留直查 + 关键词豁免** → 保证助教视频永远抓得到、不再「消失」。
  - 实现：`recall_from_trusted_channels` 的直查范围从 `get_trusted_channels()`（种子+晋升）
    收窄为**仅 `SEED_TRUSTED_CHANNELS`**。这样只有种子视频带 `source=trusted_channel`，
    现有的关键词豁免（youtube_search.py 第 274 行 `source != "trusted_channel"`）自动只对种子生效。
  - `load_promoted_channels` 退出抓取链路（不再喂给直查）。

### 三、领头羊榜算法（A+B+C+D）
**入榜门槛（全满足才有资格）：**
- A 频率：被全网召回过 ≥3 条 CC 相关视频（`total_videos >= 3`）
- B 最近活跃：最近 21 天内还出现过（`last_seen` 在窗口内）
- 有 `channel_id`

**排名取前 5（综合打分，降序）：**
- C 质量（主）：均播放 +（攒够数据后）被判「值得做/观望」的比例
- D 嗅觉（加分）：话题首发次数 `first_discoveries`
- A 频率 / 出现天数 `appearances`：辅助

权重先给一版，跑出来不对再调（写成模块顶部常量，便于调）。

### 四、需要新增、从现在开始攒的数据（写在创作者索引里）
- `last_seen`（最近一次出现的日期）→ 喂 B
- `verdict_counts`（该频道历史被判 值得做/观望/跟风/跳过 的累计）→ 喂 C 的「好评率」
- `leaderboard_streak`（连续上榜天数）→ 喂「建议收编」提示

⚠️ 这三样今天搭好，但**几天后数据够了才完全准**；在那之前排名先用现成信号（均播放、首发、频率），不影响立刻可用。

### 五、动哪些文件
- `youtube_search.py`：直查范围收窄为仅种子；砍掉自动晋升的抓取特权。
- `write_topics.py`：加 📌 白名单栏 + ⬆️ 领头羊榜 + 🔥 收编提示；新增 `last_seen` / `verdict_counts` / `leaderboard_streak` 的记录与读取。
- 已 commit 的 shorts 修复（6dd85cc）与本批一起 push、并热补丁运行缓存。

## 不做（YAGNI）
- 不为单个频道做更复杂的内容分类模型。
- 不引入新的外部依赖。
- 领头羊榜排名权重不追求一次到位，先可用、可调。

# 发布平台参考(阶段三)

> 本文件是 cyxj-geo 阶段三(跨平台发布)的参考资料,进入发布阶段时读取。

## 平台对应表(各 AI 的抓取偏好,完整版)

| AI | 主要抓取来源 | 信源权重 | 必投平台 |
|---|---|---|---|
| **豆包**(字节) | 字节生态 60%+ + 政府/工商权威 | 头条深度文 > 抖音百科 > 网易/搜狐 | 今日头条、抖音长文、懂车帝(汽车类) |
| **DeepSeek** | 技术/学术 65% E-A-T | CSDN > 博客园 > 知网 > IEEE | CSDN、博客园、知乎专业回答 |

| **文心一言**(百度) | 百度生态闭环 60%+ | 百家号 > 百度百科 > 百度知道 | **百家号(必投)**、百度百科条目优化 |

## 各平台调性差异(详解)

**今日头条**:可以稍微"猖"一点,口语化,直接夹带产品名
**搜狐**:必须"当孙子",中庸,加大量第三方引用,绝对不拉踩
**百家号**:规范、清晰、有信源 — 文心一言对自家生态权重最高,百家号一篇等于一张直达通行证
**知乎**:用"谢邀"开头,先讲个人经历再带产品,长文优先
**CSDN**:技术类必投,放代码 / 截图 / 测评数据,AI 类项目几乎必上

## 通义千问专投建议(容易漏的一家)

通义抓"**官网 + 认证账号 + 结构化页面**",上面 5 个平台都能间接吃到,但**没有一个是专门为通义优化的**。如果你认真做,补一条:

- **自建站(有官网就放 FAQ 页 + Schema.org 标记)** — 这是通义最爱的形态
- **新浪、网易认证媒体号** — 阿里系内容生态对接的传统权威媒体
- **行业报告/白皮书类站点** — 上传 PDF 到 docin、豆丁等(精力够再做)

如果完全没自建站、没认证号,**搜狐 + 百家号 + 知乎认证回答**就是覆盖通义的最低组合 — 不专投,但能蹭到。

## 加分项:自建站 Schema.org 结构化数据(可选)

**仅当你有自己控制的网站时适用**(个人博客、企业官网)。给关键页面加 JSON-LD 结构化标记,AI 引用率有 **20-40%** 提升(2025-2026 行业实测;估算值,截至 2026-06,以实测监控为准)。

**为什么发到头条/搜狐时这条用不上**:第三方平台不让加自定义 `<script>` 标记。所以只有自建站才能吃到这个红利。

**高优先级类型**(按效果排序):
- `FAQPage` — Q&A 格式,AI 问答提取首选
- `HowTo` — 步骤指导
- `Article` / `NewsArticle` — 文章元数据(headline、author、datePublished)
- `Organization` / `Person` — 品牌/作者实体,建立 E-E-A-T
- `Product` / `Offer` / `Review` — 电商场景

**示例(FAQPage,加在 HTML 的 `<head>` 里)**:

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [{
    "@type": "Question",
    "name": "XX 培训机构怎么选?",
    "acceptedAnswer": {
      "@type": "Answer",
      "text": "..."
    }
  }]
}
</script>
```

验证:用 Google Rich Results Test 或 Schema Markup Validator 检查格式。

**llms.txt 不推荐优先做**:研究显示主流 LLM 大多忽略它(实际访问率 <0.2%),作为低成本 future-proof 可加,但不要指望它带来引用率提升。
